# Proposal: Remove Authentication and Add Project Password Protection

## Problem
当前系统要求用户输入用户名登录后才能管理项目。对于一个主要在本地运行且追求便捷性的抽奖工具来说，这一步显得多余。同时，用户希望能够为特定项目设置访问密码，以保护数据隐私，而不是依赖用户账户隔离。

## Proposed Changes

### 1. 移除用户登录系统
- 删除 `LoginScreen` 组件。
- 在 `App.tsx` 中移除用户状态管理，首页直接展示项目仪表盘。
- 移除 `auth` 相关的规范文件。

### 2. 增强项目管理
- **项目数据结构更新**：在 `Project` 类型中增加可选的 `password` 字段。移除 `ownerId`，项目变为全局共享（在当前浏览器范围内）。
- **创建项目交互**：在“新建项目”模态框中增加可选的密码输入框。
- **进入项目交互**：如果项目设置了密码，在打开项目（Workspace）之前，系统必须提示用户输入密码并进行验证。

## Impact
- **用户体验**：减少了进入系统所需的步骤。增加了对特定项目的安全控制。
- **数据迁移**：现有的项目将不再与特定 `ownerId` 关联，而是全部显示在仪表盘上。
- **安全性**：从基于“用户”的隔离转变为基于“项目”的密码保护。由于完全在客户端运行，这主要防止他人直接在同一台电脑上查看敏感抽奖数据。

## Acceptance Criteria
- [ ] 应用启动后直接显示项目列表，不再出现登录界面。
- [ ] 用户可以成功创建不带密码的项目，并能直接打开。
- [ ] 用户可以成功创建带有密码的项目。
- [ ] 打开带密码的项目时，会弹出密码输入框。输入正确密码后方可进入工作区，输入错误则提示失败。
- [ ] 现有的 `storage` 工具类已更新，移除用户相关的冗余代码。
