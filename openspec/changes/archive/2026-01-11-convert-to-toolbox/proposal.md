# Proposal: Convert Project to Toolbox

## Problem
目前项目定位为单一的“年会抽奖系统 (Lucky Draw Pro)”。随着日常开发中积累的工具越来越多，需要将项目重构为一个“工具箱 (Toolbox)”应用，以便集成多种不同类型的日常工具。

## Proposed Changes

### Frontend
- **主页重构**：将现有的 Dashboard 重新设计为工具箱主界面。
- **搜索功能**：在主页右上角添加搜索框，支持对各种工具的搜索。
- **工具列表**：展示目前已有的工具（目前仅有“抽奖应用”）。
- **路由调整**：
  - `/`：工具箱主页。
  - `/lucky-draw`：现有的抽奖应用入口（原 Dashboard 逻辑，即项目管理）。
  - `/project/:id`：保持不变，用于具体的抽奖项目。

### Documentation
- **README.md**：更新项目描述，将其定义为工具箱应用，并描述其包含的日常工具。
- **openspec/project.md**：更新项目愿景和定位，从单一应用转变为工具平台。

### Structure
- **工具定义**：在前端代码中定义一套工具元数据结构，用于渲染主页列表和支持搜索。

## Impact
- **定位变化**：从特定的垂直应用转变为通用的工具箱应用。
- **可扩展性**：更方便后续添加新的独立工具，如“格式化工具”、“转换工具”等。
- **用户体验**：主页更简洁，具备搜索能力，方便用户快速定位所需工具。
